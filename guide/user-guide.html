TA Scheduler

User Guide

Getting Started

Windows Users:

  1. Download and run "TA Scheduler.exe"
  2. Prepare your three CSV files (see formats below)
  3. Drag and drop files into the application or use Browse buttons
  4. Adjust settings if needed (defaults usually work fine)
  5. Click "Run Scheduler"
  6. Review results and export if satisfied

Mac Users:

  1. Download the DMG file
  2. Double-click to mount the DMG file
  3. If you get "DMG file is damaged" error: Go to System Preferences > Security & Privacy > General, and click "Open Anyway"
  4. Drag "TA Scheduler" to your Applications folder
  5. In Applications, locate "TA Scheduler" but DON'T double-click it yet
  6. Right-click the app and select "Open" (this is important for first launch)
  7. Click "Open" when prompted about unidentified developer
  8. If blocked: Go to System Preferences > Security & Privacy > General, and click "Open Anyway"
  9. Prepare your three CSV files (see formats below)
  10. Drag and drop files into the application or use Browse buttons
  11. Adjust settings if needed (defaults usually work fine)
  12. Click "Run Scheduler"
  13. Review results and export if satisfied
Mac Security Note: Since this app isn't signed with an Apple Developer certificate, macOS will block it initially. The right-click > Open method and System Preferences adjustments are required to bypass Gatekeeper security safely.

Required Files

1. Max Availability CSV

Columns: TA Name, Hired For
Example:
John Smith, 20
Jane Doe, 15

2. Responses CSV

Columns: Timestamp, Email Address, Name, [8am to 9am], [9am to 10am], ...
Example:
7/28/2025 15:48:18, john@email.com, John Smith, "Monday, Tuesday", "", "Friday"
Note: List unavailable days in each time slot column

3. Requirements CSV

Columns: Lab Section, Day, Start, End, Required
Example:
LAB 801, Tuesday, 14, 17, 4
LAB 802, Thursday, 14, 17, 3
Important: Column headers must match exactly. Use 24-hour time format (14 = 2 PM).

Settings

  • Max Daily Hours: Maximum hours any TA can work per day (default: 8)
  • Max Labs per TA: Maximum different lab sections per TA (default: 2)
The default settings work well for most situations. Only change if you have specific requirements.

Understanding Results

  • Fully Covered: All required TAs assigned
  • Partially Filled: Some TAs assigned, more needed
  • Unfilled: No TAs available for this slot

Export Options

  • CSV Export: Creates summary files for further analysis
  • Excel Generation: Creates interactive schedules with individual TA sheets

Excel Setup (Optional)

If you choose Excel generation, follow these steps to enable automatic updates:

  1. Open the generated Excel file
  2. Press Alt+F11 to open VBA editor
  3. Insert > Module, then paste code from "vba_module_code.txt"
  4. Double-click "ThisWorkbook", paste code from "vba_thisworkbook_code.txt"
  5. Save as Excel Macro-Enabled Workbook (.xlsm)
Skip this section if you only need basic Excel files without automatic features.

Common Issues

  • Mac - "DMG file is damaged/can't be opened": Go to System Preferences > Security & Privacy > General, click "Open Anyway"
  • Mac - "Cannot open because it is from an unidentified developer": Right-click the app, select "Open", then click "Open" in the dialog
  • Mac - App blocked after right-click method: Go to System Preferences > Security & Privacy > General, click "Open Anyway" button next to the blocked app message
  • Mac - "App will damage your computer": This indicates potential quarantine attribute. Try: xattr -dr com.apple.quarantine /Applications/TA\ Scheduler.app in Terminal
  • Mac - Multiple security prompts: You may need to repeat the right-click > Open process several times for nested components
  • File upload errors: Check column headers match exactly
  • No assignments: Verify TA availability doesn't conflict with lab times
  • Poor coverage: Consider adjusting max daily hours or max labs settings
  • Missing TAs: Check that all TAs have submitted availability responses
  • Slow performance: Large datasets (100+ TAs) may take several minutes to process
Mac Users Alternative Method: If all else fails, you can temporarily disable Gatekeeper system-wide by running sudo spctl --master-disable in Terminal, then re-enable it with sudo spctl --master-enable after installation. Use this only as a last resort.
